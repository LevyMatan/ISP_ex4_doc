Data Structures
Here are the data structures with brief descriptions:
 Cfactori_args_sAll the command line argmuents after parsing them. task file name - hard copy information from argv[0] to remove dependency of shared memory. This string is going to be read by multiple threads. priority file name number of threads number of tasks: should match te number of lines in the tasks and priority files
 Clock_sThe struct has three synchornization objects and a counter: two mutexes, and one semaphore. All three will be used to protect the readers and writers to touch the shared resources at the same time
 Cqueue_elem_sAn element of FIFO queue. Hold on a generic data, here limited to a DWORD type. And a pointer to the following element in the queue. Last element is pointing to NULL
 Cqueue_sA queue object. Has pointer to last and first elements in the queue. Also hold a counter on the number of element, this is no an obligatory property, but it allows better dubging and can be useful in other cases. The queue is considered valid when count and first and last element state are matching. That is, if count equals to zero, both first and last element must point to NULL. Overwise, when count is one, both first and last element must point to the same element. Otherwise, when count is gretaer than one, each has to point to another element
