◆ QUEUE_push()

error_e QUEUE_push ( queue_t *  p_queue,
queue_elem_t  new_element 
)

Add a queue element to queue as the p_last_elem element, will also be the p_first_elem if queue was empty.

Parameters
[in]p_queue
[in]new_element
Returns
error_e

If the queue is empty, that means both p_last_elem and p_first_elem elements should point to NULL. In that case, we need update both the p_last_elem and p_first_elem to point to the new element. If the queue is not empty we want the current p_last_elem element to point to our p_last_elem element, and update the queue p_last_elem element with the new element.

So in both cases (empty or not) the queue p_last_elem pointer has to point to new element, and the count increases.
