Ex-4: Server

The server is responsible on communicating with two clients that would like to play agianst each other.
Communicating witht he client process to recieve the users inputs and send them back the result of the game.
Once opened, the serve will keep on running listening to the port number specified by the user with the command line.
It will exit only on crashes, or when process is terminated.

Inter thread communication

Two threads on top of the main thread will run in parrallel.
Each thread will communicate with another client process.
All three thread should communicate with one another, to do that, a temporary file will be created for the duration of the game and deleted when the server process closes.
The threads should lock the file for read or write access depends on useage.
The main thread is the one to Create the file, while his child thread will open an existing file only.
